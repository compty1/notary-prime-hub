## Deliverable

A single **`/mnt/documents/build_audit_findings.csv`** with columns:

```
id, category, severity, area, file, line, finding, fix, sprint, effort_min, status
```

- `id` — `BUG-0001` … `BUG-1100`
- `category` — one of the 14 above
- `severity` — `critical` / `high` / `medium` / `low`
- `area` — `theme` / `security` / `db` / `a11y` / `perf` / `ux` / `compliance` / `tests`
- `file`, `line` — exact location (or `N/A` for cross-cutting)
- `finding` — short defect description
- `fix` — concrete one-line implementation step
- `sprint` — A–N grouping for execution order
- `effort_min` — estimated minutes
- `status` — `open`

Findings are generated by automated scans (grep + AST + DB queries + scanner output) plus manually triaged items from this audit. CSV is sortable, filterable, and serves as the implementation roadmap.

## Execution approach
1. Run automated scanners (grep/awk/psql) to enumerate raw findings (bulk of 1,000).
2. Merge with security scanner + Supabase linter results.
3. Apply human-curated "fix" column with file/line precision.
4. Group into sprints A–N (security/critical first, polish last).
5. Write CSV to `/mnt/documents/build_audit_findings.csv`.
6. Surface as a downloadable artifact via `<lov-artifact>`.

After approval I will execute the CSV generation in one batch (no code changes in this round — pure audit deliverable).
